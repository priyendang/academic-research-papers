## 🔍 Project Overview

This repository is a version-controlled archive of **11 peer-reviewed research papers** published during my undergraduate degree in Information Technology Engineering at St. Francis Institute of Technology, Mumbai University.

The research spans Artificial Intelligence, Natural Language Processing, Biometric Systems, Medical Imaging, IoT-based Automation, and Computer Vision. My current professional background is in **DevOps and Site Reliability Engineering (SRE)**. Where genuine connections exist between this research and that domain — particularly around system reliability, cloud infrastructure, anomaly detection, and observability — they are called out explicitly in the [DevOps & SRE Relevance](#-devops--sre-relevance) section below.

## 🔗 DevOps & SRE Relevance

My current professional background is in **DevOps and Site Reliability Engineering**. Four of these papers connect genuinely to that domain — not by coincidence, but because they engage directly with problems of system reliability, distributed infrastructure, automated control, and operational security that SREs work on every day.

These connections are called out honestly. Papers without a real DevOps angle are not stretched to fit.

---

### 🖥️ Paper 11 — IoT Home Automation → Cloud Telemetry & Infrastructure Monitoring

**Paper:** Structuring and Design of Home Automation System Using IoT
**DevOps Connection:** `Observability` · `Cloud Infrastructure` · `Remote Monitoring` · `Alerting` · `Secure Remote Access`

This paper implements exactly what a modern monitoring stack does at small scale: sensors generate data, data is shipped to a cloud backend over Wi-Fi, the system alerts the user when anomalous activity is detected, and remote commands are executed over an authenticated, encrypted channel.

The core architecture — **sensor → agent → cloud → alert → remote action** — is the same pattern used in tools like Prometheus, Grafana, PagerDuty, and AWS CloudWatch. The paper also explicitly addresses double-encrypted remote command execution to prevent man-in-the-middle interception, which maps to secrets management and secure API access in SRE practice.

> See [`DEVOPS-NOTES.md`](./research-papers/iot/DEVOPS-NOTES.md) for a detailed breakdown.

### 📊 Paper 03 — AI + Big Data → Distributed Systems & Data Pipeline Reliability

**Paper:** Artificial Intelligence and Its Application in Classification of Big Data
**DevOps Connection:** `Distributed Systems` · `Data Pipeline Operations` · `NoSQL Infrastructure` · `Cloud-Native Architecture`

This paper directly engages with the operational challenges of running large-scale data infrastructure. It compares MongoDB-based operational Big Data systems (real-time, interactive workloads) against MapReduce/MPP-based analytical systems — the same architectural trade-offs SREs and platform engineers evaluate when designing reliable data pipelines.

The paper discusses cloud computing architectures that allow "massive computations to be run inexpensively and efficiently" — the same cost-reliability balance that drives cloud infrastructure decisions in SRE work. The operational reliability requirements of each system type (availability, throughput, failover behaviour) are concerns that sit squarely in the SRE domain.

> See [`DEVOPS-NOTES.md`](./research-papers/artificial-intelligence/DEVOPS-NOTES-bigdata.md) for a detailed breakdown.

### 🛡️ Paper 01 — AI + Intrusion Detection → DevSecOps & Network Anomaly Detection

**Paper:** Artificial Intelligence and Its Associated Applications _(Section 3.1.2 — Fuzzy Inference System in Intrusion Detection)_
**DevOps Connection:** `DevSecOps` · `Network Observability` · `Anomaly Detection` · `Security Monitoring`

Section 3.1.2 of this paper covers the use of Artificial Neural Networks and Fuzzy Inference Systems for real-time network intrusion detection. The system uses SNORT for live IP traffic analysis and packet logging, builds a signature pattern database using Protocol Analysis and Neuro-Fuzzy learning, and was validated against the 1998 DARPA Intrusion Detection Evaluation Dataset.

This is directly relevant to modern DevSecOps: anomaly-based intrusion detection, network traffic observability, and pattern-matching against known threat signatures are all live responsibilities in security-aware SRE roles. The shift from rule-based to ML-assisted anomaly detection in this paper mirrors the trajectory of modern tools like Falco, Datadog Security Monitoring, and AWS GuardDuty.

> See [`DEVOPS-NOTES.md`](./research-papers/artificial-intelligence/DEVOPS-NOTES-intrusion.md) for a detailed breakdown.

### ⚙️ Paper 05 — Power Stabilization → System Reliability & Self-Healing Automation

**Paper:** Enhancement of Power Stabilization System Using Artificial Intelligence
**DevOps Connection:** `System Reliability` · `Fault Tolerance` · `Self-Healing Systems` · `Adaptive Automation`

This is the most conceptually rich SRE connection in the repository. The paper's central problem — how do you maintain stability in a complex, nonlinear system that is continuously subject to unpredictable disturbances and fault conditions? — is exactly the problem SRE addresses for distributed software systems.

The paper's proposed solution (replacing rigid classical controllers with adaptive AI-driven controllers that detect disturbances and respond dynamically) maps precisely to modern SRE concepts: automated remediation, adaptive load balancing, circuit breakers, and self-healing infrastructure. The paper even addresses oscillation damping and cascade failure prevention — concepts that translate directly to retry storms, cascading timeouts, and backoff strategies in distributed services.

> See [`DEVOPS-NOTES.md`](./research-papers/artificial-intelligence/DEVOPS-NOTES-reliability.md) for a detailed breakdown.
